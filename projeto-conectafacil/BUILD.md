# ConectaFácil - Guia de Build e Deploy\n\n## Pré-requisitos\n\nAntes de começar, certifique-se de ter instalado:\n\n- **Node.js** (versão 18+)\n- **pnpm** (gerenciador de pacotes)\n- **Expo CLI** (para desenvolvimento e build)\n- **Android Studio** (para build Android) ou **Xcode** (para build iOS)\n\n---\n\n## Instalação de Dependências\n\n```bash\ncd /home/ubuntu/conectafacil\npnpm install\n```\n\n---\n\n## Desenvolvimento Local\n\n### Iniciar o Servidor de Desenvolvimento\n\n```bash\npnpm dev\n```\n\nIsso iniciará:\n- **Metro Bundler** (empacotador React Native)\n- **Servidor Express** (backend local)\n\nO app estará disponível em: `https://localhost:8081`\n\n### Testar em Dispositivo\n\n#### Android\n\n```bash\npnpm android\n```\n\nIsso abrirá o emulador Android ou conectará ao dispositivo físico.\n\n#### iOS\n\n```bash\npnpm ios\n```\n\nIsso abrirá o simulador iOS.\n\n#### Web\n\n```bash\npnpm dev:metro\n```\n\nAbra o navegador em `http://localhost:8081`.\n\n---\n\n## Build para Produção\n\n### Build Android APK\n\n```bash\nexpo build:android\n```\n\nOu usando EAS (Expo Application Services):\n\n```bash\nnpx eas build --platform android\n```\n\n### Build iOS\n\n```bash\nexpo build:ios\n```\n\nOu usando EAS:\n\n```bash\nnpx eas build --platform ios\n```\n\n### Build Web\n\n```bash\npnpm build\n```\n\nOs arquivos estáticos serão gerados em `dist/`.\n\n---\n\n## Configuração de Ambiente\n\n### Variáveis de Ambiente\n\nCrie um arquivo `.env` na raiz do projeto:\n\n```env\n# Exemplo de configuração\nNODE_ENV=production\nEXPO_PORT=8081\n```\n\n### App Config\n\nEdite `app.config.ts` para personalizar:\n\n- **Nome do App**: `appName`\n- **Slug**: `appSlug`\n- **Logo URL**: `logoUrl`\n- **Bundle ID** (iOS): `iosBundleId`\n- **Package Name** (Android): `androidPackage`\n\n---\n\n## Estrutura do Projeto\n\n```\nconectafacil/\n├── app/                          # Código da aplicação\n│   ├── (tabs)/\n│   │   ├── index.tsx            # Tela principal\n│   │   └── _layout.tsx          # Layout de abas\n│   └── _layout.tsx              # Layout raiz\n├── components/                   # Componentes reutilizáveis\n│   ├── screen-container.tsx     # Container seguro para tela\n│   └── ui/\n│       └── icon-symbol.tsx      # Mapeamento de ícones\n├── hooks/                        # Hooks customizados\n│   ├── use-colors.ts            # Hook de cores\n│   ├── use-jitsi-meet.ts        # Hook de Jitsi Meet\n│   └── ...\n├── lib/                          # Utilitários e configurações\n│   ├── utils.ts                 # Funções auxiliares\n│   ├── theme-provider.tsx       # Provedor de tema\n│   └── ...\n├── assets/                       # Imagens e recursos\n│   └── images/\n│       ├── icon.png             # Ícone do app\n│       ├── splash-icon.png      # Ícone da splash screen\n│       └── ...\n├── app.config.ts                # Configuração do Expo\n├── tailwind.config.js           # Configuração Tailwind\n├── theme.config.js              # Tokens de tema\n├── package.json                 # Dependências\n└── README.md                    # Documentação\n```\n\n---\n\n## Integração com Jitsi Meet\n\nO ConectaFácil usa o Jitsi Meet para videoconferência. A integração é feita através do hook `useJitsiMeet()` em `hooks/use-jitsi-meet.ts`.\n\n### Como Funciona\n\n1. Usuário digita seu nome\n2. Usuário toca no botão \"Entrar na Chamada\"\n3. Hook `useJitsiMeet()` constrói a URL do Jitsi Meet\n4. App abre a URL em navegador (Web) ou WebBrowser (Mobile)\n5. Usuário entra na sala \"familia-conecta\" com seu nome\n\n### Personalização\n\nPara mudar a sala padrão, edite `app/(tabs)/index.tsx`:\n\n```typescript\nawait joinCall({\n  userName: userName.trim(),\n  roomName: \"sua-sala-aqui\",  // Mude aqui\n});\n```\n\n---\n\n## Testes\n\n### Executar Testes\n\n```bash\npnpm test\n```\n\n### Verificar TypeScript\n\n```bash\npnpm check\n```\n\n### Linting\n\n```bash\npnpm lint\n```\n\n---\n\n## Deploy\n\n### Deploy na Expo Go (Desenvolvimento)\n\n```bash\nexpо publish\n```\n\n### Deploy em Produção\n\nUse **EAS (Expo Application Services)** para deploy automático:\n\n```bash\nnpx eas build --platform all\nnpx eas submit --platform all\n```\n\nIsso enviará o app para as lojas (Google Play Store e Apple App Store).\n\n---\n\n## Troubleshooting\n\n### Erro: \"Metro Bundler não inicia\"\n\n```bash\nrm -rf node_modules/.cache\npnpm install\npnpm dev\n```\n\n### Erro: \"Permissões de câmera/microfone\"\n\nVerifique `app.config.ts` e certifique-se de que as permissões estão configuradas:\n\n```typescript\npermissions: [\"CAMERA\", \"RECORD_AUDIO\", \"POST_NOTIFICATIONS\"]\n```\n\n### Erro: \"Jitsi não abre\"\n\nVerifique se:\n- A URL do Jitsi está correta\n- O dispositivo tem conexão com internet\n- O navegador suporta WebRTC\n\n---\n\n## Recursos Úteis\n\n- [Documentação Expo](https://docs.expo.dev/)\n- [Documentação Jitsi Meet](https://jitsi.github.io/handbook/)\n- [React Native Docs](https://reactnative.dev/)\n- [NativeWind (Tailwind para React Native)](https://www.nativewind.dev/)\n\n---\n\n## Suporte\n\nPara dúvidas ou problemas, consulte a documentação oficial ou entre em contato com o time de desenvolvimento.\n
